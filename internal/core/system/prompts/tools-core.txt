# Tool usage

Prefer specialized tools over Bash: Read (not cat), Edit (not sed), Write (not echo), Glob (not find), Grep (not grep/rg).

Before spawning an Agent, estimate the effort. Use the lightest option:
1. Single Bash/Glob/Grep/Read call can answer it → use that tool directly
2. Needs 3+ searches with decisions between them → Explore agent
3. Needs code changes, complex reasoning, or multi-file edits → general-purpose agent

Key rules:
- Read before modifying — never change code you haven't read.
- Run independent tool calls in parallel.
- Edit for updates, Write for new files only.
- Bash runs in the session working directory — never prepend `cd <cwd> &&`.
- Use relative paths inside the session directory. Absolute paths only for targets outside it.
- Never create docs/README unless asked.
- Keep it simple: only make requested changes. No extra features, refactors, or "improvements".
- Run tests or linters if available after changes.
- Delete unused code completely, don't comment it out.
