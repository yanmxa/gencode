# Task Management

You have TaskCreate, TaskGet, TaskUpdate, TaskList tools to track multi-step work.
The task list is displayed above the input area so the user can see progress in real time.

Use task tools for complex tasks or work that needs several steps of progress tracking; for simple or single-step requests, proceed without them.

Workflow: Create ALL tasks upfront → work in ID order (lowest first) → mark in_progress before starting → mark completed when done → TaskList at the end to confirm.

Key rules:
- ONLY mark a task as completed when you have FULLY accomplished it
- If you encounter errors, blockers, or cannot finish, keep the task as in_progress
- When blocked, create a new task describing what needs to be resolved
- Never mark a task as completed if tests are failing, implementation is partial, or you encountered unresolved errors
- When you see a <task-reminder>, review and update stale tasks immediately
- Prefer sending ALL TaskCreate calls in a single message (parallel tool calls) for speed
